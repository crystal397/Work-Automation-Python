======================================================================
  공기연장 간접비 보고서 자동 생성 시스템 — 사용안내
======================================================================

[ 사전 준비 ]

  1. Python 패키지 설치 (최초 1회)
     pip install -r requirements.txt

  2. Tesseract OCR 설치 (extract 명령 사용 시)
     https://github.com/UB-Mannheim/tesseract/wiki
     설치 경로: C:\Program Files\Tesseract-OCR\


======================================================================
  기본 사용법 (단일 현장)
======================================================================

  [ 경로 설정 ] 먼저 .env 파일을 메모장으로 열어 현장 경로 두 줄을 수정하세요.
    REPORT_INPUT_DIR  = 수신자료가 들어있는 폴더
    REPORT_OUTPUT_DIR = 보고서를 저장할 폴더
  설정하면 아래 모든 명령에서 경로 인수를 생략할 수 있습니다.
  (경로를 직접 넣으려면 하단 "경로 지정 방법 — 방법 B" 참고)

  Step 0.   수신자료 자동 분류 (선택 — 권장)
    python main.py classify
    → .env의 REPORT_INPUT_DIR 경로를 자동으로 사용
    → 파일명·키워드 기준으로 INCLUDE/SKIP 자동 분류
    → INCLUDE: 계약서, 공문, 조직도, 산출내역서, 하도급계약서
    → SKIP:    급여대장, 경비 영수증 (Step 2.7에서 수동 입력)

    INCLUDE 파일만 자동 복사하려면 --copy 옵션 추가:
    python main.py classify --copy
    → output/input_filtered/ 폴더에 INCLUDE 파일만 복사됨
    ※ 경로를 직접 지정하려면: python main.py classify "경로" --copy

  Step 1.   수신자료 추출 (급여대장·경비 자료 제외하고 투입)
    ┌ Step 0에서 --copy를 사용한 경우 (권장):
    │   python main.py extract --filtered
    │   → output/input_filtered/ 폴더(INCLUDE 파일만)를 자동으로 입력으로 사용
    │
    └ Step 0을 건너뛴 경우 (전체 수신자료 폴더 그대로 투입):
        python main.py extract
        → .env의 REPORT_INPUT_DIR 전체를 추출 대상으로 사용
        → 급여대장·경비 영수증도 포함되므로 extract 후 직접 제외 가능

  Step 2.   Claude Code에서 분석 요청 (구조만 추출 — 금액 제외)
    "extracted_for_analysis.md 파일을 읽고,
     분석지시서.md의 지침에 따라
     analysis_result.json 으로 저장해줘"
    → 인원명·기간·항목명·요율 추출 / 급여·경비 금액은 null

  Step 2.5. 분석지시서 결과 재검토 (권장)
    python main.py review
    → 출력된 재검토 요청문을 Claude Code에 붙여넣기

  Step 2.7. 금액 입력 (외부 확인 후)
    python main.py amounts
    → 인원별 급여·비목별 경비 입력 가이드 출력
    → 외부 사이트에서 확인한 금액을 확인 후 Claude Code에 요청:
       "analysis_result.json 에 아래 금액을 입력해줘:
        - 홍길동 월급여: 5,200,000원
        - 지급임차료 실비: 12,000,000원 ..."

  Step 3.   보고서 생성
    python main.py generate

    유형을 직접 지정하려면 --type 옵션 추가:
    python main.py generate --type A    ← 지방계약법 기반 (지자체 발주)
    python main.py generate --type B    ← 국가계약법 기반 (국가기관·공기업 발주)
    python main.py generate --type C    ← 민간·사감정 기반
    → analysis_result.json의 report_type 값을 덮어씁니다.
    → 분석 단계에서 자동 판단된 유형이 틀렸을 때 수동 수정용으로 사용하세요.

    [ Word 출력 방식 ]
    보고서_초안.docx는 유형별 레퍼런스 템플릿 기반으로 생성됩니다.
    · 템플릿(src/generator/templates/template_A/B/C.docx)이 있으면
      → 법령 인용·산정방식 설명 등 공통 보일러플레이트는 템플릿 그대로 유지
      → 공사명·기간·금액·인원·경비 등 현장별 데이터는 자동으로 삽입
    · 템플릿이 없으면 마크다운을 Word로 직접 변환 (폴백)

    ※ 템플릿 재생성이 필요한 경우 (개발자용):
       python src/generator/build_templates.py
       → src/reference/ 폴더의 13개 레퍼런스 .docx 파일이 필요합니다.


======================================================================
  Step 0 파일 자동 분류 상세
======================================================================

  수신자료를 한 폴더에 모두 넣으면 파일명·내용 키워드로 자동 분류합니다.

  분류 결과:
    INCLUDE  ★ : 파일명/내용이 구조 자료로 명확히 판단됨
    INCLUDE  ☆ : 구조 자료로 추정되나 확인 권장
    SKIP     ★ : 금액 자료로 명확히 판단됨 (extract에서 제외)
    UNKNOWN  ? : 판단 불가 — 직접 확인 후 분류

  INCLUDE 패턴 (자동 포함):
    계약서, 변경계약서, 계약통보, 도급계약, 하도급계약
    공문, 협조문, 통보문, 공기연장, 설계변경, 실정보고
    조직도, 경력증명, 재직증명, 인원현황
    산출내역, 원가계산, 내역서, 기성내역
    보험료, 요율, 산재

  SKIP 패턴 (자동 제외):
    급여, 임금, 급료, 월급, 급여대장, 급여명세
    영수증, 세금계산서, 계산서, 청구서, 전력비, 임차료

  --copy 옵션: INCLUDE 파일만 output/input_filtered/ 에 자동 복사


======================================================================
  Step 1에서 제외해도 되는 자료 (금액은 외부 확인 예정)
======================================================================

  아래 자료는 extract에 포함하지 않아도 됩니다.
  금액은 Step 2.7에서 외부 사이트를 통해 직접 입력합니다.

  제외 가능:
    - 급여대장·임금대장 (월 급여는 외부 확인)
    - 경비 영수증·세금계산서
      (노무비, 전력수도광열비, 여비교통통신비, 지급임차료,
       지급수수료, 도서인쇄비, 세금과공과, 복리후생비, 소모품비)

  반드시 포함:
    - 계약서 원본, 변경계약서 (계약금액·기간·당사자 확인용)
    - 현장조직도 또는 경력증명서 (인원명·소속·직무·기간 확인용)
    - 수발신 공문 (공기연장 귀책사유 확인용)
    - 산출내역서·공사원가계산서 (보험료율·일반관리비율·이윤율 확인용)
    - 하도급계약서 (하도급사 목록 확인용)


======================================================================
  Step 2.5 재검토를 권장하는 이유
======================================================================

  - 하도급사가 계약서 기준이 아닌 공문 언급 업체로 잘못 포함되는 경우 방지
  - 급여 없는 인원(monthly_salary=null)을 자동 탐지하여 보완 요청
  - 분석지시서.md 체크리스트 7항목 자동 확인

  review 명령은 현재 JSON 상태를 요약하고,
  Claude Code에 붙여넣을 수 있는 재검토 요청문을 즉시 출력합니다.


======================================================================
  FAIL 파일 처리 (추출 품질 개선)
======================================================================

  Step 1 완료 후 extraction_report.md 를 확인하세요.
  FAIL 파일이 있으면 포맷 변환 후 재추출:

  | 포맷      | 조치                                    |
  |-----------|----------------------------------------|
  | HWP       | 한글 → 다른 이름으로 저장 → PDF 선택    |
  | 스캔 PDF  | 300dpi 이상 재스캔 또는 전자 원본 사용  |
  | 보안 PDF  | 보안 해제 또는 인쇄 → PDF 저장          |
  | XLS       | Excel에서 xlsx로 다시 저장              |

  변환된 파일을 원본과 같은 input 폴더에 추가 후:
    python main.py extract  (기존 파일은 캐시로 스킵, 새 파일만 처리)


======================================================================
  경로 지정 방법 (현장별 폴더 분리)
======================================================================

  방법 A — .env 파일 설정 (같은 현장을 반복 작업할 때 편함)
  ─────────────────────────────────────────────────────────
  프로젝트 폴더에 .env 파일이 있습니다. 메모장으로 열어서 경로를 수정하세요.

  .env 파일 내용 예시:
  ┌──────────────────────────────────────────────────────────────┐
  │ REPORT_INPUT_DIR="D:\현장자료\송도11-1공구\수신자료"          │
  │ REPORT_OUTPUT_DIR="C:\...\output(송도)"                      │
  └──────────────────────────────────────────────────────────────┘

  설정 후에는 경로 인수 없이 명령만 입력해도 됩니다:
    python main.py extract
    python main.py generate

  현장이 바뀔 때마다 .env의 두 경로를 수정하면 됩니다.
  (.env.example 파일을 참고용 원본으로 보관하세요)


  방법 B — 커맨드라인 직접 지정 (일회성, 또는 두 현장 동시 작업 시)
  ─────────────────────────────────────────────────────────────────
  .env 설정 없이 명령마다 경로를 직접 입력합니다:

  classify: python main.py classify  "입력경로" [--copy]
  extract:  python main.py extract   "입력경로" "출력경로"
  extract:  python main.py extract   --filtered "출력경로"   ← classify --copy 후 사용
  amounts:  python main.py amounts   "출력경로"
  review:   python main.py review    "출력경로"
  generate: python main.py generate  "출력경로"
  generate: python main.py generate  "출력경로" --type A|B|C  ← 유형 직접 지정

  경로 우선순위: 커맨드라인 인수 > .env 파일 > 기본값(프로젝트/input, output/)

  REPORT_FILTERED_DIR(.env):
    classify --copy 결과 폴더를 .env에 지정할 수 있습니다.
    설정하지 않으면 REPORT_OUTPUT_DIR/input_filtered/ 를 자동 사용합니다.


======================================================================
  하도급사 간접비 자료 추가 방법
======================================================================

  하도급사로부터 급여명세서·경비 자료를 수령하면 별도 투입합니다.

  Step 1b.  python main.py subextract "하도급사명" "하도급사자료경로"
              → 출력된 Claude Code 요청문을 붙여넣기
              → Claude가 analysis_result.json의 해당 업체에 데이터 추가

  완료 후 Step 3(generate) 실행하면 하도급사 간접비 자동 산정


======================================================================
  두 현장 동시 작업 예시
======================================================================

  터미널 1 (인덕원 추출 진행 중):
    python main.py extract "D:\수신자료\인덕원" "C:\...\output(인덕원5공구)"

  터미널 2 (송도 보고서 생성):
    python main.py generate "C:\...\output(송도)"

  서로 다른 출력 폴더를 사용하므로 충돌 없음


======================================================================
  출력 파일 목록
======================================================================

  output(현장)/
    input_filtered/              ← classify --copy 결과 (INCLUDE 파일만)
    extracted_for_analysis.md   ← Claude Code 분석용 원문 (분석지시서 포함)
    extraction_report.md        ← 추출 품질 보고서 (FAIL 시 조치 방법 포함)
    analysis_result.json        ← Claude Code 작성 분석 결과
    analysis_result.json.bak    ← review 실행 시 자동 백업
    보고서_초안.md               ← 최종 보고서 (마크다운)
    보고서_초안.docx             ← 최종 보고서 (Word, 템플릿 기반)
    보고서_초안_YYYYMMDD_HHMMSS.md/.docx  ← generate 재실행 시 이전 버전 자동 백업

  프로젝트 루트/
    분석지시서.md                ← 분석 지침 (금액 제외, 하도급사·요율 판단 기준)

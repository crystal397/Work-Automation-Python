# 공기연장 간접비 분석 지시서

## 0. 금액 데이터 처리 방침 ← 반드시 먼저 읽기

**급여(monthly_salary)와 경비 금액(amount_actual/estimated)은 추출하지 않습니다.**

이 시스템의 분析 단계에서는 아래 항목만 추출합니다:
- 계약 정보, 날짜, 당사자명
- 간접노무 인원 명단 (이름·소속·직무·산정기간)
- 경비 항목명 (금액 제외)
- 보험료·일반관리비·이윤 **요율**
- 공문 이력, 변경계약 이력, 하도급사 목록

**금액은 모두 로 기재**하고, 외부 확인 후 수동 입력합니다.
( 명령으로 입력 템플릿 출력 가능)

---

이 문서는 `extracted_for_analysis.md`를 분석하여 `analysis_result.json`을 작성할 때
**반드시 준수해야 하는 지침**입니다. JSON 스키마 가이드(`[JSON 작성 가이드]`)와 함께 읽으세요.

## 1. 하도급사 목록 판단 기준 ← 가장 많이 틀리는 항목

### 포함 기준 — 다음을 **모두** 만족하는 업체만 포함

1. **하도급계약서** 또는 **하도급 내역서**에 당사자로 명시된 업체
2. 공기연장 기간 중 실제로 공사를 수행한 업체

### 제외 기준 — 아래 중 하나라도 해당하면 **반드시 제외**

| 유형 | 예시 | 이유 |
|------|------|------|
| JV(공동수급체) 구성원 | "OO건설 외 1개사 공동도급" | 원도급 당사자이므로 하도급사 아님 |
| 발주자 측 업체·기관 | 감리단, 설계사, LH, SH 등 | 계약 당사자 반대편 |
| 공문·회의록에만 언급된 업체 | 협의 대상, 민원 제기자 등 | 하도급 계약 없음 |
| 공기연장 기간에 공사가 없는 업체 | 이미 공사 완료 후 철수 | 간접비 청구 대상 없음 |

### 업체명 기재 방법

- **반드시 하도급계약서의 정식 법인명** 사용
- 약칭·통칭 사용 금지 (예: "엘엔케이시설물" → "엘엔케이종합건설(주)")
- 불확실하면 `"name": "확인 필요 — [약칭]"` 으로 기재하고 unresolved에 추가

## 2. 간접노무비 인원 추출 기준

### 포함

- **현장 상주 인원**으로 급여명세서(임금대장)에 이름·금액이 함께 기재된 인원
- 공사관리(소장·공무·공정), 안전, 품질, 측량, 사무 담당자

### 제외

- 본사 파견 인원 (현장 미상주 확인된 경우)
- 대표이사·임원 (별도 청구 기준 적용)
- 일용직 (직접노무비 항목)
- 이름만 있고 급여 금액이 전혀 없는 인원

### 급여 기재 방법

- **실제 급여명세서 또는 임금대장의 금액** (세전 기준, 원 단위 정수)
- 급여명세서가 없거나 금액 확인 불가 → `"monthly_salary": null` (절대 0 기재 금지)
- null로 기재한 경우 반드시 `unresolved`에 사유 추가

### 퇴직급여충당금

- 명세서에 퇴직급여충당금이 별도 기재된 경우 → `"retirement_rate"` 에 비율 입력
- 별도 기재 없으면 생략 (시스템 기본값 8.33% 자동 적용)

## 3. 경비 직접계상 항목 기준

### 포함 가능한 항목 (영수증·세금계산서 확인 필수)

| 항목명 | 설명 |
|--------|------|
| 지급임차료 | 사무실·창고·장비 임차 |
| 전력수도광열비 | 전기·수도·가스비 |
| 여비교통통신비 | 출장비, 통신비 (통합 기재) |
| 소모품비 | 현장 사무용품 |
| 도서인쇄비 | 도면·보고서 인쇄 |
| 지급수수료 | 보증수수료, 안전진단비 등 |

### 제외 항목 (별도 계산됨, rates에만 기재)

- 산재보험료·고용보험료 → `rates.industrial_accident`, `rates.employment`
- 국민연금·건강보험·노인장기요양보험 → **rates에 기재 불요** (법정 요율 자동 적용)

## 4. 보험료 및 요율 확인 방법

**산출내역서(공사원가계산서)** 또는 **낙찰내역서**에서 확인:

| rates 항목 | 확인 위치 | 비고 |
|-----------|----------|------|
| `industrial_accident` | 원가계산서 > 산재보험료율 | % 단위를 소수로 (예: 3.7% → 0.037) |
| `employment` | 원가계산서 > 고용보험료율 | 마찬가지 |
| `general_admin` | 원가계산서 > 일반관리비율 | |
| `profit` | 원가계산서 > 이윤율 | 최대 15% |

> ⚠️ 문서에서 확인되지 않으면 반드시 `null` 기재. 임의 추정 금지.

## 5. JV(공동수급체) 지분율 처리

계약상대자가 공동수급체(JV)인 경우:
- 에 의뢰사(간접비를 청구하는 구성원)의 지분율을 입력
- 예: A사 60%, B사 40% JV에서 A사가 의뢰한 경우 → 
- 단독 수급이면 생략 (기본값 1.0 = 전액 청구)
- JV 지분율은 계약서 또는 공동수급협정서에서 확인

---

## 6. 연장 기간 산정 기준

- `extension.start_date` = **최초 계약의 준공일 다음 날** (변경계약 기산점)
- `extension.end_date` = **최종 변경계약서의 준공일**
- `extension.total_days` = 변경계약서 차수별 연장일수 합산 (단순 날짜 차이 아님)
- 각 차수의 `extension_days`가 있으면 합산 확인

## 6. 재검토 체크리스트 — JSON 작성 후 자기검토

JSON을 작성한 뒤 아래 항목을 순서대로 확인하고, 하나라도 "아니오"이면 수정 후 저장하세요.

```
[ ] 하도급사 목록이 하도급계약서 기준인가?
    (공문·구두 언급 업체, JV 구성원 제외했는가?)

[ ] 간접노무 인원의 monthly_salary가 모두 실제 급여명세서 값인가?
    (금액 확인 불가인 경우 null로 기재했는가? 0 기재는 없는가?)

[ ] extension.start_date가 최초 준공일 다음 날인가?

[ ] rates의 요율이 모두 산출내역서에서 직접 확인한 값인가?
    (확인 안 된 요율은 null인가?)

[ ] source_totals에 소스 문서의 합계 금액을 기재했는가?
    (급여대장 합계, 경비 내역 합계 없으면 null)

[ ] unresolved에 확인하지 못한 모든 항목을 기재했는가?

[ ] 모든 _source 필드에 실제 파일명과 페이지/시트명이 있는가?
```

## 7. 소스 문서 합계 기록 (교차검증용)

`source_totals` 필드에 소스 문서에 기재된 합계 금액을 기록하세요.
계산 결과와 대조하여 추출 오류를 조기에 발견합니다.

| 항목 | 확인 위치 |
|------|----------|
| `indirect_labor_total` | 급여대장·임금대장의 "합계" 행 금액 |
| `expenses_total` | 경비 내역서의 "합계" 금액 |
| `grand_total` | 기존에 제출된 간접비 산정서가 있는 경우 최종 합계 |

- 소스 문서에 합계가 없으면 `null` (채워넣기 금지)
- 있으면 반드시 기재 — 계산값과 10% 이상 차이 시 data_checker가 경고

## 8. 불확실한 경우 처리 원칙

| 상황 | 처리 방법 |
|------|----------|
| 금액 확인 불가 | `null` 기재 + `unresolved` 추가 |
| 업체명 불확실 | `"확인 필요 — [약칭]"` + `unresolved` 추가 |
| 날짜 불확실 | `null` 기재 + `unresolved` 추가 |
| 두 문서 간 금액 상충 | 더 신뢰할 수 있는 문서 기준으로 기재, `_source`에 "OO.pdf 기준 (△△.pdf와 상이)" 명시 |
| 항목 존재 여부 불확실 | 포함하지 않고 `unresolved`에 "확인 필요" 추가 |

> **임의로 값을 추정하거나 채워 넣지 마세요.**
> 오류가 있는 값보다 null이 훨씬 낫습니다.

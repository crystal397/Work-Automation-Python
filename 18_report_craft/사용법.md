# 공기연장 간접비 보고서 자동 생성 — 사용법

---

## 준비

#### 1. 수신자료를 `input/` 폴더에 넣기

폴더 구조는 자유롭게. 하위 폴더도 모두 자동 탐색합니다.

```
input/
├── 계약서.pdf
├── 변경계약서/
│   ├── 1차변경.pdf
│   └── 2차변경.pdf
├── 급여자료/
│   └── 급여명세서.xlsx
├── 공문/
│   └── 공기연장요청.hwp
└── 영수증/
    └── 임차료.tif
```

지원 포맷: `PDF`, `Excel(.xlsx/.xls)`, `HTML`, `XML`, `HWP/HWPX`, `TIF/이미지`

> 압축 파일(.zip, .alz 등)은 자동으로 건너뜁니다.

---

## Step 1 — 텍스트 추출

터미널에서 실행:

```bash
cd "c:\Users\CT Group\Desktop\python work\18_report_craft"
python main.py extract
```

- 진행 상황과 남은 시간이 **진행바(tqdm)**로 표시됩니다.
- 처리 완료된 파일은 **캐시**에 저장되어, 중단 후 재실행 시 이미 처리한 파일은 자동으로 건너뜁니다.
- 자료를 추가한 경우 `input/`에 파일을 넣고 다시 실행하면 **새로 추가된 파일만** 처리합니다.

완료되면 `output/` 폴더에 아래 2개 파일이 생성됩니다.

| 파일 | 내용 |
|------|------|
| `output/extraction_report.md` | 파일별 추출 품질 결과 (✅ OK / ⚠️ WARN / ❌ FAIL) |
| `output/extracted_for_analysis.md` | Step 2에서 분석할 전체 텍스트 |
| `output/extract_cache.json` | 처리 완료 파일 캐시 (자동 관리) |

> ⚠️ WARN/FAIL 파일이 있으면 `extraction_report.md`를 열어 확인하세요.
> FAIL 파일은 내용을 못 읽은 것이며, 보고서에 `[출처 확인 필요]`로 표시됩니다.

---

## Step 2 — Claude Code에서 분석 요청

이 대화창(Claude Code)에서 아래 문장을 그대로 입력합니다:

> **`output/extracted_for_analysis.md` 파일을 읽고 분석해서 `output/analysis_result.json` 으로 저장해줘**

Claude Code가 파일을 읽고 아래 항목을 추출합니다:
- 보고서 유형 (A: 지방계약법 / B: 국가계약법 / C: 민간)
- 계약 정보, 변경계약 이력, 공기연장 기간
- 간접노무 인원 및 급여
- 경비 항목, 보험료·일반관리비·이윤 요율
- 수발신 공문 이력
- 확인 불가 항목 목록

완료되면 `output/analysis_result.json` 파일이 생성됩니다.

---

## Step 3 — 보고서 생성

터미널에서 실행:

```bash
python main.py generate
```

### 데이터 충족도 검사

보고서 생성 전에 **필수/권장 항목 미흡 여부를 자동으로 검사**합니다.

```
[데이터 충족도 검사]

  ❌ [간접노무비] 월 급여
       문제: 월 급여가 없는 인원: 홍길동
       조치: 급여명세서에서 해당 인원의 월 급여(원 단위)를 확인하세요.

  ⚠️  [경비] 직접계상비목
       영수증·세금계산서에서 발생 경비를 확인하세요.

필수 항목이 미흡합니다. 아래 중 하나를 선택하세요:
  1) analysis_result.json을 직접 수정하고 다시 실행
  2) Claude Code에 추가 분석 요청
  3) 미흡한 채로 보고서 초안 생성
```

- **1번**: `output/analysis_result.json`을 직접 열어 수정 후 다시 실행
- **2번**: 추가 자료를 `input/`에 넣고 Step 1부터 재실행하거나, Claude Code에 재분석 요청
- **3번**: 미흡 항목을 `[확인 필요]`로 표시한 채 보고서 생성

### 생성 파일

| 파일 | 내용 |
|------|------|
| `output/보고서_초안.md` | 마크다운 형태의 보고서 초안 |
| `output/보고서_초안.docx` | Word 문서 (바로 편집 가능) |
| `output/extraction_report.md` | 추출 품질 보고서 (Step 1에서 생성) |

---

## 주의사항

- `[확인 필요]`로 표시된 항목은 수동으로 직접 확인 후 수정하세요.
- `analysis_result.json` 내용을 직접 수정해도 됩니다. 수정 후 Step 3을 다시 실행하면 반영됩니다.
- 수신자료가 추가되면 `input/`에 파일을 추가하고 Step 1부터 다시 실행합니다. (기존 파일은 캐시로 스킵)

---

## 전체 흐름 요약

```
input/ 에 자료 넣기
    ↓
python main.py extract      → output/extracted_for_analysis.md
  (진행바 표시, 중단 시 캐시로 재개)
    ↓
Claude Code에 분석 요청      → output/analysis_result.json
    ↓
python main.py generate
  (데이터 충족도 검사 → 미흡 항목 안내 → 선택)
    ↓
output/보고서_초안.docx
```

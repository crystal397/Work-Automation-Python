# 수신자료 파일 점검 도구 (check_files.py)

수신한 폴더 내 모든 파일을 순회하며 열림 여부 및 손상 여부를 자동으로 점검하고, 결과를 엑셀 보고서와 회신 공문으로 출력하는 도구입니다.

---

## 주요 기능

### 1. 폴더 전체 순회
- 지정한 폴더 하위의 모든 파일과 폴더를 재귀적으로 탐색
- 빈 폴더도 감지하여 기록
- Windows 긴 경로(260자 초과) 및 특수문자 경로 처리 지원
- 폴더 경로 7단계까지 감지

### 2. 파일 열기 및 상태 점검
형식별로 실제 열기를 시도하여 상태를 판단합니다.

| 형식 | 점검 방식 |
|------|-----------|
| `.xlsx` `.xlsm` `.xlsb` | openpyxl로 열기 → 실패 시 xlrd 재시도 → 실패 시 바이너리 분석 |
| `.xls` | openpyxl → xlrd → 바이너리(OLE2) 순으로 시도 |
| `.docx` | python-docx로 열기, 내용 여부 확인 |
| `.pptx` | python-pptx로 열기, 슬라이드 수 확인 |
| `.pdf` | PyMuPDF로 열기, 페이지 수 확인 |
| `.zip` | 압축 해제 후 내부 파일 개별 점검 |
| `.7z` | py7zr로 압축 해제 후 내부 파일 개별 점검 |
| `.rar` | rarfile로 압축 해제 후 내부 파일 개별 점검 |
| `.msg` `.eml` | 파일 크기로 판단 (100bytes 이상이면 정상) |
| 그 외 | 파일 크기로 판단 (0bytes이면 손상) |

### 3. 비고 사유 분류
문제가 있는 파일은 비고에 아래와 같이 사유를 기록합니다.

| 비고 | 설명 |
|------|------|
| `파일 손상 (빈 파일)` | 파일 크기가 0 |
| `파일 손상 (내용 없음)` | 열리지만 내용이 없음 |
| `파일 손상 (압축 파일 오류)` | 압축 파일 구조가 손상됨 |
| `파일 손상 (파일 오류)` | 파일 구조 오류 |
| `열기 실패 (암호 보호)` | 암호가 설정된 파일 |
| `열기 실패 (보안 문서: AIP/DRM 보호)` | Microsoft AIP 또는 IRM 보안 적용 |
| `열기 실패 (보안 문서: IRM/DRM 보호)` | OLE2 형식의 IRM 보안 적용 |
| `열기 실패 (접근 권한 없음)` | 파일 접근 권한 부족 |
| `열기 실패 (로그인 필요)` | 인증이 필요한 파일 |
| `열기 실패 (경로 오류: 특수문자 등)` | 파일명 특수문자로 인한 경로 오류 |
| `열기 실패 (형식 불일치 또는 손상)` | 확장자와 실제 형식이 다름 |
| `폴더 비었음` | 해당 폴더에 파일 없음 |

### 4. 압축 파일 내부 점검
- `.zip` / `.7z` / `.rar` 파일을 임시 폴더에 해제 후 내부 파일 개별 점검
- 압축 내 한글 파일명 인코딩 자동 처리 (UTF-8 / CP949 / EUC-KR)
- 점검 완료 후 임시 폴더 자동 삭제
- 엑셀에서 압축파일명을 상위 분류로, 내부 파일명을 파일명 컬럼에 표시

### 5. 엑셀 보고서 생성 (`수신자료_파일점검결과_YYYYMMDD_HHMMSS.xlsx`)
두 개의 시트로 구성됩니다.

**시트 1: 수신자료 정리**
- 번호 / 대분류 / 중분류 / 소분류 / 세부분류 / 파일명 / 파일형식 / 비고
- 폴더 계층 구조를 분류 컬럼에 반영
- 비고 있는 행: 빨간 배경 강조
- 빈 폴더 행: 노란 배경 강조
- 하단 집계: 총 파일 수 / 비고 기록 건수

**시트 2: 비고 기록 파일 목록**
- 비고가 기록된 파일만 별도 시트에 모아 표시

## 실행 방법

### Python으로 직접 실행
```bash
python check_files.py
```

### exe 파일로 실행
```
check_files.exe
```

실행 시 폴더 선택 창이 표시됩니다. GUI 사용이 불가한 환경에서는 콘솔에서 경로를 직접 입력합니다.

---

## 출력 파일

| 파일 | 설명 |
|------|------|
| `수신자료_파일점검결과_*.xlsx` | 전체 점검 결과 엑셀 보고서 |

출력 파일은 **스크립트 실행 위치(현재 디렉토리)** 에 저장됩니다.

---

## 필요 라이브러리 설치

```bash
pip install openpyxl python-docx python-pptx PyMuPDF py7zr rarfile xlrd
```

> `.rar` 파일 점검 시 **UnRAR** 실행 파일이 별도로 필요합니다.  
> 설치되지 않은 경우 `.rar`는 크기로만 판단합니다.

---

## exe 빌드 방법

`check_files_build.ps1`을 `check_files.py`와 같은 폴더에 놓고 실행합니다.

```
우클릭 → PowerShell로 실행
```

빌드 완료 후 `dist/check_files.exe` 단일 파일만 배포하면 됩니다.

---

## 점검 한계 사항

| 형식 | 한계 |
|------|------|
| `.hwp` `.hwpx` | 한글 라이브러리 미지원 → 크기로만 판단 |
| `.doc` `.ppt` | 구형 Office 형식 → 크기로만 판단 |
| 암호 걸린 정상 파일 | 열기 실패로 기록되나 실제로는 정상 파일일 수 있음 |
| 내용이 깨진 파일 | 라이브러리가 오류 없이 열리면 정상으로 판단될 수 있음 |

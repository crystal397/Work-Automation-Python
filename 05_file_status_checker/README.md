# 수신자료 파일 점검 도구 (check_files.py)

수신한 폴더 내 모든 파일을 순회하며 열림 여부 및 손상 여부를 자동으로 점검하고, 결과를 엑셀 보고서로 출력하는 도구입니다.

---

## 주요 기능

### 1. 폴더 전체 순회
- 지정한 폴더 하위의 모든 파일과 폴더를 재귀적으로 탐색
- 빈 폴더도 감지하여 기록
- Windows 긴 경로(260자 초과) 및 특수문자 경로 처리 지원
- **폴더 깊이(depth)를 실제 구조에서 자동 계산** — 압축 파일 내부 폴더 구조까지 반영

### 2. 파일 열기 및 상태 점검
형식별로 실제 열기를 시도하여 상태를 판단합니다.

#### 열어서 내용 검사하는 형식
| 형식 | 점검 방식 | 파일형식 표시 예시 |
|------|-----------|-------------------|
| `.xlsx` `.xlsm` `.xlsb` | openpyxl → xlrd → 바이너리(OLE2) 순으로 시도 | Microsoft Excel 워크시트 |
| `.xls` | 동일 | Microsoft Excel 워크시트 |
| `.docx` | python-docx로 열기, 내용 여부 + 텍스트 깨짐 비율 검사 | Microsoft Word 문서 |
| `.pptx` | OLE2 시그니처 감지 → IRM/DRM 스트림 확인, 정상 ZIP이면 python-pptx로 슬라이드 수 + 텍스트 깨짐 비율 검사 | Microsoft PowerPoint 프레젠테이션 |
| `.pdf` | PyMuPDF로 열기, 텍스트량으로 스캔본/텍스트 구분, AIP/DRM 감지, 한국형 PDF DRM(Markany·SyncEZ 등) 안내 페이지 패턴 감지 | Adobe Acrobat 문서 (텍스트) / (스캔본) |
| `.hwp` | `.pfile` 시그니처(RMS/AIP 암호화) 감지, OLE2 구조·FileHeader·BodyText/Section0 스트림 검증, HWP ParaText 레코드 직접 파싱으로 텍스트 추출 후 깨짐 비율 검사 | 한글 문서 |
| `.hwpx` | ZIP 구조 확인, Contents/section*.xml 파싱 → `<hp:t>` 텍스트 추출 후 깨짐 비율 검사 | 한글 문서 |
| `.png` `.jpg` `.jpeg` `.gif` `.bmp` `.tif` `.tiff` | Pillow로 유효성 확인, 해상도·색상모드 추출 | PNG 이미지 (1920×1080, RGB) |
| `.dxf` | ezdxf로 파싱(설치 시), 미설치 시 SECTION/ENDSEC 텍스트 시그니처 확인 | AutoCAD DXF 도면 (1,234개 객체) |
| `.dwg` | 파일 시그니처 `AC####` 확인으로 버전 판별 | AutoCAD 도면 (AC1032) |
| `.dwl` `.dwl2` | 텍스트로 읽어 내용 확인 | DWL 파일 |
| `.xer` | ERMHDR 시그니처 또는 XML 파싱 | XER 파일 |
| `.txt` `.log` | UTF-8 → CP949 → UTF-16 순으로 인코딩 시도 | LOG 파일 |
| `.csv` `.rtf` | 동일 | CSV 파일 |
| `.mp4` `.mov` | 파일 시그니처(`ftyp`, `moov` 등) 확인 | MP4 동영상 |
| `.avi` | 파일 시그니처 `RIFF....AVI` 확인 | AVI 동영상 |
| `.mp3` | 파일 시그니처 `ID3` 또는 `FF FB` 확인 | MP3 오디오 |
| `.bak` `.tmp` | ZIP/OLE/PDF/텍스트 시그니처로 원본 형식 추정 | OLE 기반 백업 (doc/xls 등) |
| `.zip` | 압축 해제 후 내부 파일 개별 점검 | 압축(ZIP) 파일 |
| `.7z` | py7zr로 압축 해제 후 내부 파일 개별 점검 | 7-Zip 압축 파일 |
| `.rar` | rarfile로 압축 해제 후 내부 파일 개별 점검 | RAR 압축 파일 |
| `.msg` `.eml` | 파일 크기로 판단 (100bytes 이상이면 정상) | Outlook 항목 |

#### OLE 구조 검증 형식
| 형식 | 점검 방식 |
|------|-----------|
| `.doc` | OLE2 구조 검증, WordDocument 스트림 존재 확인, IRM/DRM 스트림 감지 |
| `.ppt` | OLE2 구조 검증, PowerPoint Document 스트림 존재 확인, IRM/DRM 스트림 감지 |

#### 크기만 확인하는 형식
| 형식 |
|------|
| `.docm` `.pptm` (구형 Office 매크로 바이너리) |
| `.dwf` `.mpp` `.xml` `.json` |
| 등록되지 않은 기타 확장자 |

### 3. 압축 파일 내부 점검
- `.zip` / `.7z` / `.rar` 파일을 임시 폴더에 해제 후 내부 파일 개별 점검
- 압축 내 한글 파일명 인코딩 자동 처리 (UTF-8 / CP949)
- **압축 내부 폴더 구조를 depth 열에 직접 배분** — 내부 경로가 파일명 컬럼에 뭉치지 않음
- 점검 완료 후 임시 폴더 자동 삭제

### 4. 비고 사유 분류
문제가 있는 파일은 비고에 아래와 같이 사유를 기록합니다.

| 비고 | 설명 |
|------|------|
| `파일 손상 (빈 파일)` | 파일 크기가 0 |
| `파일 손상 (내용 없음)` | 열리지만 내용이 없음 |
| `파일 손상 (압축 파일 오류)` | 압축 파일 구조가 손상됨 |
| `파일 손상 (OLE 구조 오류)` | .doc/.ppt/.hwp OLE2 구조 손상 |
| `파일 손상 (WordDocument 스트림 없음)` | .doc OLE 내 핵심 스트림 누락 |
| `파일 손상 (PowerPoint Document 스트림 없음)` | .ppt OLE 내 핵심 스트림 누락 |
| `파일 손상 (HWP FileHeader 없음)` | .hwp OLE 내 FileHeader 스트림 누락 |
| `파일 손상 (HWP 본문 없음)` | .hwp OLE 내 BodyText/Section0 스트림 누락 |
| `파일 손상 (HWP 본문 압축 오류)` | .hwp BodyText 스트림 압축 해제 실패 |
| `파일 손상 (HWPX 본문 없음)` | .hwpx ZIP 내 section XML 파일 없음 |
| `파일 손상 (HWPX XML 오류)` | .hwpx XML 파싱 실패 |
| `내용 깨짐 의심 (깨진 문자 XX%)` | 텍스트 깨짐 비율 15% 이상 (.docx/.pptx/.pdf/.hwp/.hwpx) |
| `파일 손상 (이미지 오류)` | 이미지 파일 구조 손상 |
| `파일 손상 (DWG 시그니처 오류)` | DWG AC#### 시그니처 없음 |
| `파일 손상 (DXF 구조 오류)` | DXF SECTION/ENDSEC 구조 없음 |
| `파일 손상 (XER 구조 오류)` | XER 시그니처 및 XML 구조 없음 |
| `파일 손상 (파일 오류)` | 기타 파일 구조 오류 |
| `열기 실패 (암호 보호)` | 암호가 설정된 파일 |
| `열기 실패 (보안 문서: AIP/DRM 보호)` | Microsoft AIP 또는 IRM 보안 적용 |
| `열기 실패 (보안 문서: IRM/DRM 보호)` | OLE2 형식의 IRM 보안 적용 |
| `열기 실패 (접근 권한 없음)` | 파일 접근 권한 부족 |
| `열기 실패 (로그인 필요)` | 인증이 필요한 파일 |
| `열기 실패 (경로 오류: 특수문자 등)` | 파일명 특수문자로 인한 경로 오류 |
| `열기 실패 (형식 불일치 또는 손상)` | 확장자와 실제 형식이 다름 |
| `열기 실패 (시그니처 불일치)` | 동영상/오디오 파일 시그니처 오류 |
| `열기 실패 (인코딩 불명)` | 텍스트 파일 인코딩 판별 실패 |
| `폴더 비었음` | 해당 폴더에 파일 없음 |

### 5. 엑셀 보고서 생성 (`수신자료_파일점검결과_YYYYMMDD_HHMMSS.xlsx`)
두 개의 시트로 구성됩니다.

**시트 1: 수신자료 정리**
- 번호 / N단계(폴더 계층) / 파일명 / 파일형식 / 비고
- **폴더 계층 열 수가 실제 depth에 맞게 자동 조정** (압축 내부 포함)
- 비고 있는 행: 빨간 배경 강조
- 빈 폴더 행: 노란 배경 강조
- 하단 집계: 총 파일 수 / 비고 기록 건수

**시트 2: 비고 기록 파일 목록**
- 비고가 기록된 파일만 별도 시트에 모아 표시

## 실행 방법

### Python으로 직접 실행
```bash
python check_files.py
```

### exe 파일로 실행
```
check_files.exe
```

실행 시 폴더 선택 창이 표시됩니다. GUI 사용이 불가한 환경에서는 콘솔에서 경로를 직접 입력합니다.

---

## 출력 파일

| 파일 | 설명 |
|------|------|
| `수신자료_파일점검결과_*.xlsx` | 전체 점검 결과 엑셀 보고서 |

출력 파일은 **스크립트 실행 위치(현재 디렉토리)** 에 저장됩니다.

## 필요 라이브러리 설치

```bash
pip install openpyxl python-docx python-pptx PyMuPDF olefile py7zr rarfile xlrd tqdm Pillow
```

#### 선택 설치 (미설치 시 해당 형식은 크기 또는 시그니처만 확인)
```bash
pip install ezdxf     # DXF 객체 수 파싱
```

> `.rar` 파일 점검 시 **UnRAR** 실행 파일이 별도로 필요합니다.  
> 설치되지 않은 경우 `.rar`는 크기로만 판단합니다.

---

## exe 빌드 방법

`check_files_build.ps1`을 `check_files.py`와 같은 폴더에 놓고 실행합니다.

```
우클릭 → PowerShell로 실행
```

빌드 완료 후 `dist/check_files.exe` 단일 파일만 배포하면 됩니다.

## 점검 한계 사항

| 형식 | 한계 |
|------|------|
| `.hwp` | OLE2 기반 HWP는 ParaText 레코드 파싱으로 텍스트 깨짐 검사. `.pfile` RMS/AIP 암호화 감지. 단, HWP 비압축 모드 파일은 깨짐 검사 불가 |
| `.doc` `.ppt` | OLE 구조 및 핵심 스트림 존재 확인, IRM 감지. 내부 텍스트 추출은 불가 |
| `.docm` `.pptm` | 구형 Office 매크로 형식 → 크기로만 판단 |
| `.dwg` | 시그니처(버전 코드) 확인만 가능, 내부 도면 데이터 검증 불가 |
| 암호 걸린 정상 파일 | 열기 실패로 기록되나 실제로는 정상 파일일 수 있음 |
| 내용이 깨진 파일 | 라이브러리가 오류 없이 열리면 정상으로 판단될 수 있음 |

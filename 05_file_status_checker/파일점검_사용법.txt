================================================================================
  수신자료 파일 점검 프로그램 사용법
================================================================================

1. 프로그램 개요
--------------------------------------------------------------------------------
  지정한 폴더 내의 모든 파일을 자동으로 순회하면서 다음 항목을 점검합니다.

    - 파일 열림 여부 (열기 실패, 접근 권한, 암호 보호 등)
    - 파일 손상 여부 (빈 파일, 내용 없음, 형식 불일치 등)
    - 보안 문서 여부 (IRM/DRM/AIP 보호 문서)

  압축 파일(.zip, .7z, .rar)은 내부 파일까지 개별적으로 점검합니다.
  점검 결과는 엑셀 보고서와 회신문(Word)으로 자동 생성됩니다.


2. 사용 순서
--------------------------------------------------------------------------------
  [STEP 1] 폴더 선택
    - 프로그램 실행 시 폴더 선택 창(GUI)이 나타납니다.
    - 점검할 파일들이 들어있는 최상위 폴더를 선택합니다.
    - GUI가 실행되지 않는 환경에서는 자동으로 콘솔 입력 모드로 전환되며,
      폴더 경로를 직접 입력할 수 있습니다.

  [STEP 2] 자동 점검
    - 선택한 폴더의 모든 하위 폴더와 파일을 자동으로 순회합니다.
    - 터미널에 진행 상황이 표시됩니다.
    - 압축 파일은 임시 폴더에 해제한 뒤 내부 파일을 개별 점검합니다.

  [STEP 3] 결과 확인
    - 점검이 완료되면 아래 결과물이 생성됩니다.

      (a) 수신자료_파일점검결과_[날짜시간].xlsx  → 엑셀 보고서

    - Enter를 누르면 프로그램이 종료됩니다.


3. 결과물 상세 설명
--------------------------------------------------------------------------------
  (1) 엑셀 보고서 (.xlsx)
      - [수신자료 정리] 시트: 전체 파일 목록 (폴더 계층별 분류 포함)
      - [비고 기록 파일 목록] 시트: 문제가 있는 파일만 별도 정리
      - 각 파일에 대해 번호, 대/중/소/세부분류, 파일명, 파일형식, 비고가 기록됩니다.
      - 문제 파일은 빨간색 배경으로 강조 표시됩니다.


4. 점검 가능 파일 형식
--------------------------------------------------------------------------------
  Office 문서   : xlsx, xls, xlsm, xlsb, docx, doc, pptx, ppt, csv
  PDF 문서      : pdf
  한글 문서     : hwp, hwpx
  이미지        : png, jpg, jpeg, gif, bmp, tif, tiff
  압축 파일     : zip, 7z, rar


5. 비고(오류) 유형 안내
--------------------------------------------------------------------------------
  비고 메시지                         의미
  ---------------------------------- -----------------------------------------
  파일 손상 (빈 파일)                 파일 크기가 0바이트
  파일 손상 (내용 없음)               파일은 열리나 내용이 없음
  파일 손상 (파일 오류)               파일이 손상되어 읽을 수 없음
  파일 손상 (빈 압축파일)             압축 파일 내부에 파일이 없음
  열기 실패 (암호 보호)               비밀번호가 설정된 파일
  열기 실패 (보안 문서: IRM/DRM 보호) Microsoft IRM/DRM 보안이 적용된 문서
  열기 실패 (보안 문서: AIP/DRM 보호) Azure Information Protection 보안 문서
  열기 실패 (접근 권한 없음)          파일 접근 권한이 없음
  열기 실패 (형식 불일치 또는 손상)    확장자와 실제 파일 형식이 다름
  폴더 비었음                        하위 파일이 없는 빈 폴더


6. 참고 사항
--------------------------------------------------------------------------------
  - 점검 중 원본 파일은 수정되지 않습니다. (읽기 전용으로 열람)
  - 폴더 구조가 최대 4단계(대/중/소/세부분류)까지 엑셀에 반영됩니다.
  - Windows 환경에서 긴 경로(260자 초과)도 자동으로 처리됩니다.
  - 결과 파일은 프로그램 실행 위치(현재 디렉토리)에 저장됩니다.

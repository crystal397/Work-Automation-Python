### **🏗️ 시스템 아키텍처 및 자동화 프로세스**

본 프로젝트는 단순한 매크로를 넘어 **데이터 보안(Security)**, **예외 처리(Error Handling)**, **작업 연속성(State Persistence)**을 최우선으로 설계되었습니다.

#### **1. 전략적 사이트 구조 대응 (Dynamic Navigation)**

* **구조적 유연성**: 대상 시스템의 9개 독립 카테고리를 `CATEGORIES` 맵핑 테이블로 추상화했습니다. 각 메뉴의 `ID`를 동적으로 참조하여 클릭 로직을 공통화함으로써, 향후 사이트 UI 변경 시에도 코드 수정 없이 설정값 변경만으로 대응이 가능한 **유지보수성**을 확보했습니다.
* **보안 인증 레이어**: `python-dotenv`를 도입하여 민감한 계정 정보와 URL을 코드에서 분리했습니다. 이를 통해 협업 및 오픈소스 공개 시 발생할 수 있는 보안 취약점을 근본적으로 차단했습니다.

#### **2. 엔지니어링 기반 워크플로우 (Data Pipeline)**

단순 업로드 업무를 4단계의 체계적인 데이터 파이프라인으로 재구성했습니다.

1. **정제 (Pre-processing)**: 정규표현식 기반의 파일명 파싱을 통해 `날짜_내용_금액` 규칙에 따른 자동 분류 및 데이터 정합성을 검증합니다.
2. **변환 (Document ETL)**: `win32com`을 활용한 **이기종 문서 통합 파이프라인**을 구축했습니다. Excel 데이터를 별도의 사용자 개입 없이 시스템 표준인 PDF 포맷으로 실시간 렌더링하며, 출력 옵션(FitToPages)을 프로그래밍적으로 제어합니다.
3. **최적화 (Intelligent Chunking)**: 시스템의 9MB 업로드 제한을 극복하기 위해 **지능형 페이로드 최적화** 로직을 구현했습니다. `pypdf`로 파일 용량을 분석하고, 원본 데이터 손실 없이 최적의 용량 단위로 자동 분할하여 전송 성공률을 극대화했습니다.
4. **전송 및 모니터링 (Load & Recover)**: Selenium 기반의 가상 브라우저를 통해 일괄 업로드를 수행하며, **JSON 기반 상태 추적 시스템**을 통해 오류 발생 시 중단 지점부터 즉시 재개(Resume)할 수 있는 안정성을 제공합니다.

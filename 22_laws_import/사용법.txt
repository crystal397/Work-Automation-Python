================================================================================
  입찰공고일 기준 법령 자동 매칭 시스템 — 사용 설명서
  (공기연장 보고서 작성용 법령 버전 조회 도구)
================================================================================


▌ 이 프로그램이 하는 일
────────────────────────────────────────────────────────────────────────────────
공기연장 보고서 작성 시, 계약의 입찰공고 시점에 실제로 시행 중이던
법령 버전을 자동으로 찾아주는 도구입니다.

단순히 현행 법령이 아니라, 입찰공고일 당시 시행 중이던 법률·시행령·시행규칙
버전을 법제처 공식 데이터 기준으로 매칭하고, 조문 원문을 Word 파일로
저장해 줍니다.

  대상 법령 (15개):
  ┌───────────────────────────────────────────────────────────┐
  │ [법령] 국가계약법 / 시행령 / 시행규칙                    │
  │ [법령] 지방계약법 / 시행령 / 시행규칙                    │
  │ [법령] 조달사업에 관한 법률 / 시행령                     │
  │ [법령] 하도급법 / 시행령                                  │
  │ [행정규칙] 공사계약일반조건                               │
  │ [행정규칙] 예정가격 작성기준                              │
  │ [행정규칙] 정부 입찰·계약 집행기준                       │
  │ [행정규칙] 지방자치단체 입찰 및 계약집행기준             │
  │ [행정규칙] 산업안전보건관리비 계상 및 사용기준           │
  └───────────────────────────────────────────────────────────┘


▌ 시작 전 준비 — 법제처 기관코드(OC) 발급
────────────────────────────────────────────────────────────────────────────────
프로그램을 사용하려면 법제처 국가법령정보 공동활용 서비스의
기관코드(OC)가 필요합니다. 아래 절차로 1회만 발급하면 됩니다.

  1. https://open.law.go.kr 접속
  2. 회원가입 후 '활용 신청' 메뉴에서 활용 승인 신청
  3. 담당자 검토 후 1~2일 이내 승인
  4. 가입 시 사용한 이메일 ID가 기관코드(OC)로 사용됩니다
     (예: hong@company.com → OC는 hong@company.com)

  ※ 기관코드는 회사 담당자에게 문의하면 기존 발급된 코드를 공유받을 수 있습니다.


▌ 인터넷 연결 필수
────────────────────────────────────────────────────────────────────────────────
법제처 서버에서 실시간으로 법령 정보를 조회합니다.
인터넷이 연결되지 않은 환경에서는 캐시에 저장된 이전 조회 결과만
사용할 수 있습니다.

================================================================================
  사용 방법 (단계별)
================================================================================

STEP 1. 기관코드 입력
────────────────────────────────────────────────────────────────────────────────
화면 왼쪽 상단 '법제처 기관코드(OC)' 입력란에 발급받은 이메일 ID를 입력합니다.
(입력값은 보안을 위해 ●●●로 표시됩니다)

  - 처음 한 번 입력하면 다음 실행 시 자동으로 불러옵니다.
    (.env 파일이 프로그램과 같은 폴더에 있는 경우)


STEP 2. 입찰공고일 선택
────────────────────────────────────────────────────────────────────────────────
달력 위젯에서 해당 계약의 입찰공고일을 선택합니다.

  - 달력이 보이지 않으면 날짜를 YYYY-MM-DD 형식으로 직접 입력합니다.
    예) 2022-03-15


STEP 3. 법령 선택
────────────────────────────────────────────────────────────────────────────────
조회할 법령의 체크박스를 선택합니다.
기본값은 15개 전체 선택 상태입니다.

  - [전체 선택] 버튼: 모든 법령 선택
  - [전체 해제] 버튼: 모든 법령 해제
  - 특정 법령만 선택하면 조회 시간이 단축됩니다.


STEP 4. 매칭 실행
────────────────────────────────────────────────────────────────────────────────
[▶ 매칭 실행] 버튼을 클릭합니다.

  - 조회 중에는 진행 바가 채워집니다.
  - 법령 1개당 보통 3~10초 소요됩니다 (네트워크 상태에 따라 다름).
  - 이전에 조회한 적 있는 법령은 캐시에서 즉시 불러옵니다 (7일간 유효).
  - 하단 상태바에 현재 처리 중인 법령명이 표시됩니다.


================================================================================
  결과 확인
================================================================================

▌ [매칭 결과] 탭
────────────────────────────────────────────────────────────────────────────────
조회된 법령별 요약 표가 나타납니다.

  컬럼 설명:
  ┌────────────┬──────────────────────────────────────────────────────────┐
  │ 법령명     │ 조회한 법령 이름                                         │
  │ 공포번호   │ 해당 버전의 공포번호 (예: 대통령령 제33000호)           │
  │ 시행일     │ 해당 버전의 실제 시행 날짜                               │
  │ 부칙 경과규정│ 부칙에서 경과규정이 탐지된 경우 표시                  │
  │ 비고       │ 추가 확인이 필요한 경우 사유 표시                        │
  └────────────┴──────────────────────────────────────────────────────────┘

  행(row) 색상 의미:
  ┌──────────┬───────────────────────────────────────────────────────────┐
  │ (기본)   │ 정상 매칭 완료                                            │
  │ 빨간색   │ 부칙 경과규정 탐지 — 이전 버전 적용 여부 확인 필요       │
  │ 주황색   │ 행정규칙 연혁 조회 불가 — 현행 버전 표시 (수동 확인 필요)│
  │ 노란색   │ 자동 확정 불가 — 사용자가 직접 버전을 확인해야 함        │
  └──────────┴───────────────────────────────────────────────────────────┘

  ※ 결과 행을 클릭하면 [조문 상세] 탭에 해당 법령의 조문이 표시됩니다.


▌ [조문 상세] 탭
────────────────────────────────────────────────────────────────────────────────
[매칭 결과] 탭에서 법령 행을 클릭하면 해당 법령의 공기연장 관련 조문 원문이
이 탭에 표시됩니다.

  표시 내용:
  - 법령명, 공포번호, 시행일, 법제처 원문 URL
  - 경고 메시지 (부칙 경과규정, 연혁 조회 실패 등)
  - 공기연장 관련 키워드가 포함된 조문 원문 (조·항·호 단위)

  ※ 부칙 경과규정이 탐지된 경우 빨간색으로 경과규정 내용이 먼저 표시됩니다.
     탐지 유형:
     - 유형 A: 법령 전체에 이전 버전 적용 ("종전의 규정에 따른다")
     - 유형 B: 특정 조문에만 이전 버전 적용 ("최초 공고 분부터 적용")


▌ [실행 로그] 탭
────────────────────────────────────────────────────────────────────────────────
API 호출 과정, 버전 선정 근거, 경고 메시지 등 상세 처리 기록입니다.
문제 발생 시 이 로그를 참고하거나 담당자에게 전달하세요.


================================================================================
  Word 리포트 저장
================================================================================

[💾 Word 리포트 생성 (.docx)] 버튼을 클릭하면 저장 위치를 선택하는 창이 열립니다.

  저장 파일 구성:
  ┌────────────────────────────────────────────────────────────────────────┐
  │  표지        — 입찰공고일, 작성 일시                                  │
  │  요약 테이블 — 전체 법령 매칭 결과 한눈에 보기                        │
  │  법령별 상세 — 기본정보 / 부칙 경고 / 공기연장 관련 조문 원문         │
  │  검토 필요 목록 — 수동 확인이 필요한 항목 모아보기                    │
  └────────────────────────────────────────────────────────────────────────┘

  ※ 조문 원문은 법제처 API 응답 원문 그대로 저장됩니다. AI가 수정·요약하지 않습니다.
  ※ 매칭 실행 전에는 저장 버튼이 동작하지 않습니다.


================================================================================
  주의사항
================================================================================

  1. 부칙 경과규정 표시 시 반드시 직접 확인하세요.
     빨간색으로 표시된 법령은 "입찰공고일 기준으로 이전 버전이 적용될 수 있음"을
     의미합니다. 자동 탐지 결과를 참고로만 사용하고, 법제처 원문 URL에서 부칙을
     반드시 직접 확인하세요.

  2. 행정규칙(회계예규·고시)은 연혁 조회가 제한될 수 있습니다.
     주황색으로 표시된 항목은 연혁 데이터를 가져오지 못해 현행 버전을 표시한
     것입니다. 법제처(www.law.go.kr)에서 해당 행정규칙의 과거 이력을
     직접 검색하여 확인하세요.

  3. 캐시 유효 기간은 7일입니다.
     같은 법령을 7일 내에 다시 조회하면 저장된 결과를 사용합니다.
     최신 데이터가 필요한 경우 실행 로그에서 캐시 히트 여부를 확인하고,
     cache 폴더의 laws.db 파일을 삭제하면 캐시가 초기화됩니다.

  4. 법령 조문은 AI가 생성하지 않습니다.
     모든 조문 원문은 법제처 공식 API에서 받은 데이터입니다.


================================================================================
  오류가 발생한 경우
================================================================================

  ■ "법제처 기관코드(OC)가 없습니다" 오류
    → 기관코드를 입력하지 않았거나 잘못 입력했습니다.
       open.law.go.kr 가입 이메일 ID를 정확히 입력하세요.

  ■ "법령 MST 조회 실패" 메시지
    → 해당 법령명으로 검색 결과가 없거나 API 응답이 없는 경우입니다.
       인터넷 연결을 확인하고, [실행 로그] 탭의 오류 내용을 담당자에게 전달하세요.

  ■ 조문 내용이 비어 있음
    → 해당 법령 버전에서 공기연장 관련 키워드가 포함된 조문이 없는 경우입니다.
       법제처 원문 URL에서 직접 확인하세요.

  ■ 화면이 열리지 않거나 프로그램이 실행되지 않는 경우
    → Windows Defender 또는 백신 프로그램이 차단하는 경우가 있습니다.
       파일을 마우스 우클릭 → "속성" → "차단 해제" 체크박스를 선택하거나,
       백신 예외 경로로 등록 후 재실행하세요.

  ■ 실행 로그 파일 위치
    → 프로그램 폴더 내 logs\laws_import.log 파일에서 상세 기록을 확인할 수 있습니다.


================================================================================
  파일 구성 안내
================================================================================

  실행파일과 같은 폴더에 아래 파일·폴더가 자동으로 생성됩니다.

  ┌──────────────────────────────────────────────────────────────────────┐
  │ LawMatcher.exe     ← 실행 파일 (더블클릭으로 실행)                  │
  │ .env               ← 기관코드 저장 파일 (자동 생성)                  │
  │ cache\             ← API 조회 결과 캐시 저장 폴더 (자동 생성)        │
  │ logs\              ← 실행 로그 저장 폴더 (자동 생성)                  │
  └──────────────────────────────────────────────────────────────────────┘

  ※ .env 파일에 기관코드가 저장됩니다. 타인과 공유하지 마세요.
  ※ cache, logs 폴더는 삭제해도 프로그램 실행에 문제없습니다 (재실행 시 재생성).

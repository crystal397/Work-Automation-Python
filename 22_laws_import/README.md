# 입찰공고일 기준 법령 자동 매칭 시스템

공공 건설공사의 공기연장 보고서 작성 시, **입찰공고일 시점에 실제로 시행 중이던 법령 버전**을 자동으로 매칭하고 관련 조문 원문을 추출하는 도구입니다.

---

## 주요 기능

| 기능 | 설명 |
|---|---|
| 입찰공고일 기준 버전 매칭 | 법령 연혁을 조회하여 입찰공고일에 시행 중인 버전 자동 선정 |
| 6단계 시행일 판단 로직 | 연혁조회 → 1차 후보 → 부칙 경과규정 탐지 → 2차 후보 → 정합성 확인 → 사용자 검토 |
| 부칙 경과규정 탐지 | 법령 전체 경과규정(유형 A)·조문 단위 경과규정(유형 B) 각각 탐지 및 경고 |
| 공기연장 관련 조문 필터 | 사전 정의된 키워드 26개로 관련 조문 자동 추출 |
| 원문 직접 인용 | 법제처 API 응답 원문만 사용 — AI 재작성 없음 |
| 출처 명시 | 법령명·공포번호·시행일·조문번호 5요소 기록 |
| Word 리포트 저장 | 매칭 결과·조문 원문을 `.docx`로 출력 |
| 로컬 SQLite 캐시 | API 호출 결과 7일 캐시, 반복 조회 속도 향상 |
| 로그 파일 기록 | API 호출·버전 선정 과정·사용자 선택 전체 감사 추적 |

## 시스템 구성

```
22_laws_import/
├── main.py          ← 진입점 (로깅 초기화 → GUI 실행)
├── gui.py           ← Tkinter GUI (좌측: 입력 패널 / 우측: 결과 탭 + Word 저장)
├── engine.py        ← 핵심 매칭 엔진 (6단계 시행일 판단 로직)
├── api_client.py    ← 법제처 국가법령정보 API 클라이언트 (경로 A)

├── .env.example     ← 환경변수 템플릿
├── requirements.txt
└── requirements.md  ← 요구사항 정의서
```

## 사전 준비

### 1. 법제처 API 기관코드 발급

[https://open.law.go.kr](https://open.law.go.kr) 에서 회원가입 후 활용 승인 신청(1~2일 소요).
승인 후 가입 이메일 ID가 기관코드(OC)로 사용됩니다.

### 2. 환경변수 설정

```
# 22_laws_import/.env
LAW_API_OC=법제처_가입_이메일_ID
```

`.env.example`을 `.env`로 복사 후 발급받은 기관코드를 입력합니다.

## 실행 방법

```bash
cd 22_laws_import
pip install -r requirements.txt
python main.py
```

GUI가 열리면:
1. **기관코드** 입력 (법제처 가입 이메일 ID)
2. **입찰공고일** 선택 (달력 위젯)
3. **조회할 법령** 선택 (전체 또는 개별 선택)
4. **실행** 버튼 클릭
5. 결과 탭에서 매칭 결과·조문 원문 확인
6. **Word 저장**으로 `.docx` 리포트 출력

---

## 6단계 시행일 판단 로직

```
1. 연혁 조회     → 대상 법령의 전체 버전(공포번호·시행일) 수집
2. 1차 후보      → 시행일 ≤ 입찰공고일 중 가장 최근 버전 선정
3. 부칙 경과규정 → 1차 후보 부칙 파싱
                   유형 A (법령 전체): "이 법 시행 전에 공고된 입찰에 대해서는 종전의 규정에 따른다"
                   유형 B (조문 단위): "제○조의 개정규정은 시행 후 최초 공고 분부터 적용"
4. 2차 후보      → 유형 A: 직전 버전 전환 / 유형 B: 해당 조문만 직전 버전 병기
5. 정합성 확인   → 법률·시행령·시행규칙 시행일 불일치 시 경고
6. 사용자 검토   → 자동 확정 불가 시 두 후보 모두 출력 + 확인 요청
```

## 할루시네이션 방지 원칙

| 원칙 | 내용 |
|---|---|
| P1 원문 직접 인용 | 법령 조문은 법제처 API 원문만 사용, AI 재작성 없음 |
| P2 버전 추적성 | 법령명·공포일·공포번호·시행일·조번호 5요소 기록 |
| P3 결정 근거 기록 | 버전 선정 이유(입찰공고일·시행일 비교·부칙 검토 결과) 로그 저장 |
| P4 불확실성 표시 | 자동 확정 불가 시 두 후보 모두 출력 + 사용자 확인 요청 |
| P5 교차 검증 | API 응답 공포번호·시행일을 로컬 캐시와 교차 검증 |

## 주의사항

- **법제처 API 기관코드**는 `.env`로 관리합니다. 소스코드에 직접 입력하지 마세요.
- 행정규칙(회계예규·고시) 중 일부는 API에서 조회되지 않을 수 있습니다. 누락 시 결과에 표시됩니다.
- 부칙 경과규정 자동 해석은 패턴 매칭 기반이므로 탐지된 경우 반드시 사용자가 직접 확인하세요.
- API 호출 기록은 `logs/laws_import.log`에 저장됩니다.

# Pipeline Guide — 공기연장 간접비 보고서 매뉴얼

> 케이스 유형: NEW ✓ (전 단계 진행) / RERUN △ (정정 단계만) / RETEST ⊘ (회귀만 — 04·07·09)
> 케이스 분기 자세히: 프로젝트 루트 [CLAUDE.md](../CLAUDE.md) "케이스 분기" 섹션

새 공기연장 간접비 보고서 프로젝트를 처음부터 끝까지 진행하기 위한 단계별 가이드.

기반: 7개 검증 프로젝트 경험 (의뢰인 보고서 산정값 100% 일치 케이스).

    ↓
[05] 청구 범위 결정 → calculation_target
    ↓

### 자동화 수준 요약

- **완전 자동 (✓)**: 02 인덱싱 / 08 법령 — 명령 한 줄로 결과 생성, 사람 개입 최소
- **반자동 (△)**: 03 OCR / 04 계약 메타 / 06 사건 분석 / 07 산정 / 09 합철 — 시스템이 후보·계산을 제공, **사람이 검증·결정·선별**
- **주로 사람 작업 (⊘)**: 00 자료 정리 / 01 사전 체크 / 05 청구 범위 — 도구는 보조, **사람 판단이 핵심**

> ⚠ **사람 결정·검토가 필요한 핵심 지점**:
> 1. **05 청구 범위** — 어느 차수를 청구할지 (산정액 전체를 좌우)
> 2. **06 사건 후보 승인** — AI 가 도출한 사건 중 어떤 사건으로 진행할지
> 3. **09 첨부 큐레이션** — 자동 분류된 수천 건 중 핵심 수백 건 선별 (가장 큰 사람 작업)
> 4. **04 계약 메타 보강** — 자동 추출 70% + 대표자·입찰공고일·요율 등 사람 보강 30%

## 🎯 핵심 원칙

1. **단일 진실원** — `contract_meta.yaml` 가 모든 정보의 원천
2. **출처 의무** — 모든 값에 `_source` (file·page·raw_text·method)
3. **격식체 본문** — ~합니다 / ~하였습니다 (평서체 금지)
4. **양 끝 포함 일수** — `(end - start).days + 1`
5. **이전 baseline 활용** — 동일 의뢰인 이전 보고서 참고

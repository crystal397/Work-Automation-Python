# 📍 대용량 주소 데이터 지오코딩 및 공간 분석 파이프라인
> **20,000건 이상의 주소 데이터를 고속 좌표 변환하고, 다각적 시각화(Heatmap/Cluster)를 통해 비즈니스 인사이트를 도출하는 자동화 도구입니다.**

## 1. 프로젝트 개요
실무에서 발생하는 대규모 주소 리스트를 지리 정보로 변환하고, 이를 시각화하여 밀집 지역 분석 및 특정 거점 기준의 반경 내 분포 확인을 목적으로 합니다. 단순 시각화를 넘어 **대용량 데이터 처리 성능 최적화**와 **API 효율성**에 초점을 맞췄습니다.

## 2. 주요 기능 및 기술 스택
- **Python & Pandas**: 대규모 CSV 데이터 로드 및 전처리
- **Kakao Local API**: 주소의 위/경도 좌표 정밀 추출
- **Parallel Processing (`ThreadPoolExecutor`)**: 멀티스레딩을 통한 지오코딩 속도 최적화 (순차 대비 약 10배 이상 향상)
- **Folium Interactive Mapping**: 
  - **Heatmap**: 전체적인 밀집도(Hot-spot) 분석
  - **MarkerCluster**: 2만 건 이상의 개별 위치를 성능 저하 없이 시각화
  - **Multi-Radius Analysis**: 중심지 기준 250m, 500m, 1km 반경 가이드 라인 제공

## 3. 핵심 문제 해결 경험 (Technical Challenges & Solutions)

### ✅ 대용량 데이터 시각화 성능 최적화
- **문제점**: 20,000개의 마커를 웹 지도에 직접 렌더링할 경우 브라우저 멈춤 및 메모리 과부하 발생.
- **해결책**: `MarkerCluster`를 도입하여 인접 데이터를 그룹화하고, 줌 레벨에 따라 동적으로 상세 데이터를 노출함으로써 웹 렌더링 성능을 90% 이상 개선하였습니다.

### ✅ API 호출 비용 및 시간 절감 (Caching)
- **문제점**: 주소 데이터 내 중복 주소가 많아 불필요한 API 중복 호출이 발생하고, 일일 할당량(Quota) 초과 위험이 있음.
- **해결책**: `address_cache` 딕셔너리를 이용한 인메모리 캐싱(In-memory Caching) 로직을 구현하여 동일 주소에 대해서는 API 호출 없이 즉시 좌표를 반환하도록 설계하였습니다.

### ✅ 주소 정제 로직 고도화 (Regex)
- **문제점**: 아파트 동/호수나 괄호 안의 상세 정보 등으로 인해 API 검색 성공률이 떨어짐.
- **해결책**: 정규표현식(`re`)을 활용하여 지오코딩에 불필요한 정보를 제거하는 전처리 파이프라인(`clean_address`)을 구축하여 검색 성공률을 극대화하였습니다.

### ✅ 지연 시간(Latency) 대응을 위한 병렬 처리
- **문제점**: 2만 건을 하나씩 처리할 경우 약 2시간 이상의 소요 시간 예상.
- **해결책**: `ThreadPoolExecutor`를 활용하여 네트워크 I/O 바운드 작업인 API 요청을 병렬화(15개 스레드)함으로써 전체 처리 시간을 10분 내외로 단축하였습니다.

## 4. 데이터 파이프라인 구조
1. **Raw Data Load**: `raw_addresses.csv` 로드
2. **Preprocessing**: 정규표현식을 통한 주소 정제 및 중복 제거(Caching)
3. **Geocoding**: 카카오 API 병렬 호출을 통한 위경도 좌표 획득
4. **Data Persistence**: 좌표가 포함된 결과물을 CSV로 중간 저장 (Checkpoint)
5. **Visualization**: 최적화된 HTML 인터랙티브 지도 생성
